Step 1 — Project Overview & Goal

This step defines the project objective, inputs/outputs, and the end-to-end workflow used to convert a color-coded wind map into wind speed and wind power density on a 20×20 grid.

Project goal Why this matters

Wind maps are informative but primarily visual. This project demonstrates a reproducible method to extract that visual information into numerical data suitable for analysis and comparison.

  • Convert colors → wind speeds by calibrating to the on-screen legend.
  • Sample spatially using a consistent 20×20 grid across the region.
  • Quantify energy potential via wind power density, enabling offshore vs onshore comparisons.

Key insight

Wind power density scales with v³. Small differences in wind speed can produce large differences in energy potential, which is why offshore regions often stand out.

Inputs & outputs Scope

Input artifact (single screenshot must include):

  • Study area: California + offshore Pacific region of interest
  • Wind overlay: color-coded wind layer enabled
  • Legend: visible wind-speed colorbar (mph) used for calibration

Project outputs generated by the analysis pipeline:

  • 4-panel figure: original image, average-color grid, wind speed map, power density map
  • Summary CSV: min / mean / max values (wind speed + power density)
  • Interpretation: offshore vs onshore patterns + caveats (labels/overlays)

Workflow overview End-to-end pipeline

The workflow is implemented in Python (NumPy + Matplotlib + PIL) and can be run via a simple UI that supports screenshot selection and drag-box legend capture.

1
Capture a wind map screenshot Use Zoom Earth wind layer; keep the legend visible in the same image.
2
Calibrate using the legend Select the legend region to map RGB color values to wind speed (mph).
3
Grid sampling (20×20) Compute average RGB per grid cell to represent local wind color robustly.
4
Compute wind power density Convert mph → m/s, then compute power density per cell using P = ½ ρ v³.
5
Generate figures + summary tables Export a 4-panel visualization and min/mean/max statistics for reporting.
