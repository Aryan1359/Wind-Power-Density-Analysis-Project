What is Zoom Earth used for? Data Layer

Zoom Earth provides an interactive wind visualization layer. For this project, the wind map acts as a visual dataset: each pixel color corresponds to a wind speed value defined by the on-screen legend.

  • Wind map → provides spatial wind patterns over California and offshore waters.
  • Color legend → provides the calibration reference (color ↔ wind speed).
  • Screenshot → freezes a single moment in time so the same inputs can be analyzed reproducibly.

Why we use a screenshot Reproducibility

The wind layer is dynamic (it changes with time and user interaction). A screenshot is treated as a static dataset snapshot, allowing the analysis to be repeated and audited.

  • Deterministic input: the analysis pipeline is fully reproducible when the same image is used.
  • Shareable: the screenshot can be stored in GitHub alongside code and outputs.
  • Traceable: the legend captured in the same image documents the mapping assumptions.

Screenshot requirements Quality Control

For the wind-speed extraction to work reliably, the screenshot must preserve both the map region and the wind-speed legend without ambiguity. Small capture mistakes (cropping, blur, overlays) can introduce systematic errors in wind speed and power density estimates.

Must be visible

  • Entire study area (California + offshore region of interest)
  • Wind legend scale (colorbar with labeled wind speeds)
  • Clean wind colors (avoid heavy transparency filters)
  • Consistent zoom (avoid perspective distortions from partial captures)

Avoid if possible

  • Text overlays covering the map (labels can be interpreted as low wind or invalid pixels)
  • Legend cropped (breaks the RGB → wind speed calibration)
  • Motion blur / compression (blends colors and shifts RGB averages)
  • UI panels obscuring the region (menus, popups, tooltips)

Why screenshot quality matters

In this project, wind speed is inferred from color. If the screenshot compresses or blends colors, the RGB values measured in each grid cell drift away from the legend reference colors. That drift propagates to wind speed and then to wind power density.

Because wind power density scales with v³, small wind-speed errors can amplify into large power-density errors.

Capture workflow (what the user does) Practical Steps

The workflow below is the minimal, repeatable process used before running the analysis UI.

1
Open Zoom Earth and enable the wind layer Set the map view so the target region is fully visible. Ensure the wind color overlay is active.
2
Confirm the legend is visible and readable The legend is the calibration reference. It must remain on screen when the screenshot is taken.
3
Take a high-resolution screenshot Prefer native OS screenshots (not a compressed messenger export). Save as PNG when possible.
4
Run the project UI and select inputs Choose the screenshot file, then draw a drag-box around the legend to capture the color scale.
